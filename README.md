# Do Well to Do Good

A mental wellness app to get you back to your best — so you can make the world better again. **Designed from the ground up for privacy.**

Journal your thoughts, feelings, and mood — then save them privately or *"shout into the wind"* and let them go. Not sure what you're feeling? A gentle, fully keyboard-navigable **emotion wheel** helps you find the word. On each entry you get a mental-wellness tip drawn from **vetted, cited sources**.

---

## Features

- **Journaling** — write freely, log your mood, and either save an entry or "shout into the wind" (released and cleared, never stored).
- **Emotion wheel** — the full three-ring feelings wheel (130 feelings, each with a plain-language definition). Tap a color to zoom into a family; fully operable by mouse, touch, and keyboard (arrow keys + Enter), with a live definition panel.
- **Cited tips** — wellness tips matched to your emotion, paraphrased in plain language and **credited with a link** to the original source (NHS, NIMH/NIH, APA, Mayo Clinic, UC Berkeley's Greater Good Science Center).
- **Guest mode** — use everything without an account; nothing is saved anywhere.
- **Privacy-first** — see [Privacy & architecture](#privacy--architecture) below.
- **Installable (PWA)** with a soothing, bubble-inspired design and WCAG 2.2 AA-minded accessibility.

## Tech stack

- **[Blazor WebAssembly](https://learn.microsoft.com/aspnet/core/blazor/) (.NET 8)** — runs entirely in the browser.
- **[Tailwind CSS v4](https://tailwindcss.com/)** — styling (compiled by the .NET build).
- **[Supabase](https://supabase.com/)** — auth + database *(planned; see roadmap)*.
- **Self-hosted [Quicksand](https://fonts.google.com/specimen/Quicksand)** font — no third-party font CDN.

## Getting started

```bash
git clone https://github.com/drummerboydibs/do-well-to-do-good.git
cd do-well-to-do-good
dotnet run --project src/DoWellToDoGood
```

Then open the URL printed in the console.

> **Note on styles:** The compiled stylesheet `wwwroot/css/app.css` is generated by Tailwind at build time and is intentionally **not committed** (it's a build artifact). You don't need to do anything extra — an MSBuild target runs `npm install` and compiles the CSS automatically on `dotnet build` / `dotnet run`. For live style editing, run the watcher in a second terminal:
>
> ```bash
> cd src/DoWellToDoGood
> npm run watch:css
> ```

## Privacy & architecture

Privacy is the core design principle, not an afterthought:

- **Guest / "shout into the wind"** content stays in the browser, in memory only, and is wiped on submit, navigation, or session end — it never touches the network.
- **Saved entries (planned)** will use **zero-knowledge encryption**: each entry is encrypted in the browser with a key derived from a passphrase only the user holds, so the server stores nothing but ciphertext — readable by no one, not even a database administrator.
- **No third-party tracking** — no analytics that profile you, and no third-party font CDNs (Quicksand is self-hosted), so visiting the app doesn't leak your data to outside services.
